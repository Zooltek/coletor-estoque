# Contexto do Projeto

Sempre preservar a arquitetura existente.

Nunca alterar regras de negócio.

Priorizar:

1. Performance.

2. UX.

3. Baixo acoplamento.

4. Componentes reutilizáveis.

5. Scanner desacoplado.

---

## Objetivo

Transformar o projeto em um coletor profissional.

Sem alterar o fluxo atual de inventário.

---

## Scanner

Sempre utilizar ScannerService.

Nunca acessar CameraX diretamente pela interface.

---

## UI

Interface semelhante a coletores Zebra.

Sempre manter:

- câmera aberta
- scanner contínuo
- painel inferior
- feedback visual

---

## Nunca

Não duplicar código.

Não criar componentes grandes.

Não colocar lógica de negócio em páginas.
